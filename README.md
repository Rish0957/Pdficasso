# 📄 Pdficasso

**Pdficasso** is a simple and efficient tool for splitting and merging PDF files, designed to make PDF manipulation fast, intuitive, and hassle-free.

---

## ✨ Features

- **Merge PDFs** — Upload multiple PDF files and combine them into a single document in any order you choose.
- **Split PDFs** — Upload a PDF and extract specific pages by entering comma-separated page numbers.
- **50MB Size Limit** — Enforced on both the client and server to keep things fast and safe.
- **Drag & Drop** — Simply drag your files into the browser to get started.
- **Dark Mode UI** — A sleek, modern interface built with React and Tailwind CSS.

---

## 🛠️ Tech Stack

| Layer      | Technology                          |
| ---------- | ----------------------------------- |
| Frontend   | React (Vite), TypeScript, Tailwind CSS |
| Backend    | Node.js, Express, TypeScript        |

## 🚀 Getting Started

### Prerequisites

Make sure you have the following installed and available in your PATH:

- [Node.js](https://nodejs.org/) (v18+)
- [Git](https://git-scm.com/)

### Quick Start (Windows)

Simply **double-click** `start_prod.bat` in the project root. It will:

1. Kill any processes on ports 3001 and 5173.
2. Install dependencies for both backend and frontend.
3. Build the TypeScript backend and the Vite frontend.
4. Start both services in separate terminal windows.

Then open your browser at **http://localhost:5173**.

### Manual Start

```bash
# Backend
cd backend
npm install
npx tsc
node dist/index.js
# Runs on http://localhost:3001

# Frontend (in a new terminal)
cd frontend
npm install
npm run build
npx serve -s dist -l 5173
# Runs on http://localhost:5173
```

### Development Mode

```bash
# Backend (hot-reload)
cd backend
npm run dev

# Frontend (hot-reload, in a new terminal)
cd frontend
npm run dev
```

## 📡 API Endpoints

| Method | Endpoint      | Description                      | Body                                |
| ------ | ------------- | -------------------------------- | ----------------------------------- |
| GET    | `/`           | Health check                     | —                                   |
| POST   | `/api/merge`  | Merge multiple PDFs              | `multipart/form-data` field: `pdfs` |
| POST   | `/api/split`  | Extract pages from a single PDF  | `multipart/form-data` fields: `pdf`, `pages` (comma-separated, 0-indexed) |

---

## 🔒 Security

- **No data is stored on the server.** All PDF processing happens in-memory and is discarded after the response is sent.
- **50MB upload limit** is enforced at both the browser (frontend) and server (multer) levels.
- Sensitive files (`.env`, `node_modules`, `dist`) are excluded from version control via `.gitignore`.
